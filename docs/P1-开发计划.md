## 一、需求文档

### 项目概要

| 项 | 内容 |
|:--|------|
| 项目名称 | Hermes v0.2.0 |
| 一句话 | P1 阶段：补全 OWASP MCP Top 10 覆盖，新增 Fuzz 引擎 + 策略引擎 + 审计链 + 等保预设 |
| 为什么做 | P0 已覆盖 10 条规则（静态8+动态8），但 OWASP MCP Top 10 仍有 MCP01/02/03/04/05/06/08/09 共 8 项空白需要填补 |

### 用户

| 用户类型 | 需求 |
|----------|------|
| MCP Server 开发者 | 在开发阶段发现安全漏洞，修复建议 |
| 企业安全团队 | 按 OWASP Top 10 标准审计，生成合规报告 |
| CI/CD 流程 | 自动化安全门控，退出码集成 |
| 等保合规团队 | 等保 2.0 二级合规检查 |

### 必须做的功能（P1 核心范围）

按全网调研确定的优先级排列：

#### 模块 1：新扫描规则 SC-11/SC-12/SC-14（覆盖 MCP01）

P1 新增 3 条规则（SC-09/10/13/15 为 P2）：

| ID | 规则名 | 严重级 | 检测内容 |
|:--|------|:--:|------|
| SC-11 | env-secret-leak | High | env 字段暴露敏感值 |
| SC-12 | sensitive-file-args | Medium | 启动参数传递 .env/*.pem/*.key |
| SC-14 | unsafe-filesystem | High | 文件系统 Server 访问根目录 `/` 或 `~` |

**P2 预留（架构预埋，P1 不实现）：** SC-09 (no-timeout)、SC-10 (unpinned-package)、SC-13 (missing-description)、SC-15 (supply-chain-risk)

#### 模块 2：审计链 AU-01~04（覆盖 MCP08）

| ID | 功能 | 说明 |
|:--|------|------|
| AU-01 | HMAC-SHA256 审计链 | `Hn = HMAC(Hn-1, record_n)` |
| AU-02 | 审计链文件 | `.hermes/chain-{command}-{timestamp}.json` |
| AU-03 | 审计链验证 | `hermes verify` 命令 |
| AU-04 | 审计记录结构 | 时间戳、规则ID、严重级别、证据、修复建议 |

#### 模块 3：策略引擎 PL-01~03

| ID | 功能 | 说明 |
|:--|------|------|
| PL-01 | JSON 策略文件解析 | 读取 `.hermes-policy.json` |
| PL-02 | 严重级别阈值 | `min_severity` 过滤 |
| PL-03 | 规则开关 | 按 ID 启用/禁用 |

**P2 预留（架构预埋，P1 不实现）：** PL-04 (白名单)、PL-05 (basic/strict/enterprise 预设，仅 dengbao 提前至 P1)

#### 模块 4：新探测规则 PR-08/PR-10/PR-13（覆盖 MCP05/07）

P1 新增 3 条规则（PR-09/11/12 为 P2）：

| ID | 规则名 | 严重级 | 检测内容 |
|:--|------|:--:|------|
| PR-08 | ssrf-probe | Critical | 参数注入内网 URL 检测 |
| PR-10 | session-predictability | High | Session ID 格式/自增检测 |
| PR-13 | path-traversal | High | 路径穿越检测（`../../../` 等） |

**P2 预留（架构预埋，P1 不实现）：** PR-09 (ssrf-redirect)、PR-11 (session-replay)、PR-12 (session-fixation)

> 等保预设 SM-05（`--preset dengbao`）随策略引擎模块实现，编译进二进制。

#### 模块 5：Fuzz 引擎 FZ-01/02/03/04/08（覆盖 MCP03/05）

P1 新增 4 条 Fuzz 测试（FZ-05/06/07 为 P2）。FZ-01 为 P0 欠债，需在 P1 开始时立即补上。

| ID | 测试 | 严重级 | 输入 |
|:--|------|:--:|------|
| FZ-01 | empty-input | High | `""`, `null`, `{}`（⚠️ P0 欠债） |
| FZ-02 | oversized-input | Medium | 1MB 字符串 |
| FZ-03 | special-chars | Medium | `\x00`, `\u0000` 等控制字符 |
| FZ-04 | path-injection | High | `../../../` 路径 |
| FZ-08 | crash-detect | High | 汇总：检测 5xx/连接断开/超时 |

**P2 预留（架构预埋，P1 不实现）：** FZ-05 (sql-injection)、FZ-06 (command-injection)、FZ-07 (prompt-injection)

#### 模块 6：新 CLI 命令与标志

| 命令/标志 | 适用命令 | 说明 |
|:--|------|------|
| `hermes report <path>` | 独立命令 | 读 JSON 结果文件 → 渲染格式化报告（终端/HTML） |
| `--format html` | audit,probe,report | HTML 格式输出 |
| `--policy <file>` | audit,probe,fuzz | 外部策略文件（随策略引擎） |
| `--preset dengbao` | audit,probe,fuzz | 等保预设（basic/strict/enterprise 为 P2） |
| `--min-severity <level>` | audit,probe,fuzz | 最低显示级别（随策略引擎） |
| `--audit-key <file>` | audit,probe,verify | HMAC 密钥文件（随审计链） |
| `--init-key` | audit | 交互式密钥创建（随审计链）

### 特殊情况处理

| 情况 | 处理 |
|------|------|
| 策略文件格式错误 | 报错并提示正确的 JSON 格式示例 |
| 审计密钥文件不存在 | 提示用 `--init-key` 创建 |
| 策略文件与 preset 同时指定 | 报错：互斥 |
| Fuzz 测试目标服务器宕机 | 记录崩溃，继续下一个测试 |
| 等保预设引用的规则未实现 | 优雅跳过，不报错（P2 规则在 P1 阶段未实现） |

## 二、设计方案

### 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 策略文件格式 | JSON（`.hermes-policy.json`） | `serde_yaml_ng` 维护停滞，改用 JSON 零风险 |
| 新 CLI 命令 | `hermes fuzz <url>` + `hermes verify <file>` + `hermes report <file>` | 新增 3 个 Subcommand |
| 策略引擎 | 独立模块 `src/policy/` | 解耦，可单独测试 |
| 审计链 | 独立模块 `src/chain/` | 复用 FRP-X 审计链逻辑 |
| Fuzz 引擎 | 独立模块 `src/fuzz/` | 与 probe 共享 HTTP 客户端 |
| 等保预设 | 内置枚举 + 仅 dengbao | 编译进二进制，零文件依赖。basic/strict/enterprise 为 P2 |
| HTML 报告 | `--format html` 作为输出格式 | 自包含 HTML 内联 CSS，零外部依赖 |

### 阶段 B：审计链（模块 2）

#### 任务 B1：创建 chain 模块骨架 + AU-04 类型定义

**依赖**：无
**干什么**：创建 `src/chain/` 目录，定义 `AuditRecord`、`AuditChain` 类型

**验收标准**：
-   `src/chain/mod.rs` + `src/chain/types.rs` 创建完毕
-   类型定义完整（含 serde 序列化），`cargo check` 通过

#### 任务 B2：实现 AU-01 HMAC 链构建

**依赖**：任务 B1
**干什么**：实现 `Hn = HMAC-SHA256(Hn-1, serialize(record_n))`
-   用 `sha2` + `hmac` 库（已在依赖中）
-   支持从密钥文件读取 HMAC 密钥
-   每次 audit/probe 结束后自动生成 chain 文件

**验收标准**：
-   `cargo test` 中 chain 模块 ≥3 个测试（正常链、空链、篡改检测）
-   chain 文件输出到 `.hermes/chain-{command}-{timestamp}.json`

#### 任务 B3：实现 AU-03 verify 命令

**依赖**：任务 B2
**干什么**：新增 `hermes verify <audit-file>` 子命令
-   重算 HMAC 链
-   检测篡改
-   输出验证报告

**验收标准**：
-   `hermes verify` 能检测到篡改的 chain 文件
-   `--audit-key` 标志可用
-   退出码正确（0=有效, 2=被篡改, 1=错误）

#### 任务 B4：实现 --init-key 交互式密钥创建

**依赖**：任务 B2
**干什么**：`--init-key` 标志，引导用户创建 16+ 字节随机密钥

**验收标准**：
-   `hermes audit --init-key` 创建密钥文件
-   支持 `HERMES_AUDIT_KEY` 环境变量
