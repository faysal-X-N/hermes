## 一、需求文档

### 项目概要

| 项 | 内容 |
|:--|------|
| 项目名称 | Hermes v0.3.0 |

## 三、设计方案

### 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| GitHub Action | 复合 Action（`action.yml`） | 无需 Docker，直接调 Hermes 二进制 |
| SARIF 格式 | 手动构建 JSON（不引库） | SARIF 结构简单，省依赖 |
| 管理级 HTML | 纯 CSS 图表（bar + donut chart） | 零外部依赖，自包含 |
| 等保预设编译进二进制 | 零文件依赖，与 REQUIREMENTS.md 一致 | 2026-06-02 |
| `make_finding` | Builder 模式 | 每个字段一个 setter |

### 模块变更

```
新增:
  action.yml                    # GitHub Action 入口
  src/report/sarif.rs           # SARIF 格式输出

修改:
  src/audit/rules.rs            # +4 条新规则（SC-09/10/13/15）
  src/audit/types.rs            # Builder for Finding
  src/probe/*.rs                # +6 条新探测
  src/fuzz/engine.rs            # +3 条新 Fuzz
  src/policy/*.rs               # +白名单 + 预设 + 基线对比
  src/report/html.rs            # 管理级图表增强（`--format html-management`）
   src/chain/hmac.rs             # 审计链构建
   src/report/*.rs               # 报告输出
   src/main.rs                   # +--fix + SARIF + policy init
  Cargo.toml                    # 新增 `glob` crate 替换手写匹配器
```

### 格式说明

| 标志 | 用途 | P1 已有? |
|:--|------|:--:|
| `--format html` | 技术级报告（规则/证据/修复建议表格） | ✅ |
| `--format html-management` | 管理级报告（图表/趋势/合规打分） | P2 新增 |
| `--format sarif` | GitHub Code Scanning 格式 | P2 新增 |

## 四、任务清单

**总共 22 个任务，预计工期 2 周。**

### 阶段 TD：基础重构（最优先，模块 8）

> 先做重构，让后续所有新规则用 Builder + 统一评分，避免写完再返工。

#### 任务 TD1：make_finding → Builder 模式

**依赖**：无
**干什么**：
- 消除 9 参数 `make_finding`（clippy 警告）
- `Finding::builder().rule_id(...).severity(...).build()`

**验收标准**：
- clippy `too_many_arguments` 从 5 降至 4
- 所有 54 现有测试仍通过

#### 任务 TD2：compute_score 去重

**依赖**：无
**干什么**：
- main.rs 的 `compute_probe_score` 和 types.rs 的 `compute_score` 统一为 `audit::types::compute_score`

**验收标准**：
- 只有一个 `compute_score` 函数
- 所有测试通过

#### 任务 TD3：glob crate 替代手写匹配器

**依赖**：无
**干什么**：
- Cargo.toml 添加 `glob` crate

**验收标准**：
- 扫描结果与替换前一致
- ≥5 个现有 glob 测试仍通过

### 阶段 SF：SARIF + --fix（模块 3）

> 先做 SARIF，因为 GitHub Action 依赖它。

#### 任务 SF1：实现 SARIF 输出格式

**依赖**：无
**干什么**：
- 创建 `src/report/sarif.rs`
- 按 SARIF v2.1.0 规范构建 JSON
- `hermes audit . --format sarif`
- 每个 Finding → 一条 `result`（ruleId、message、locations、level）

**验收标准**：
- 生成有效 SARIF JSON
- ≥2 个测试

#### 任务 SF2：实现 --fix 自动修复

**依赖**：无
**干什么**：
- `hermes audit . --fix` → 自动修复可修项
- 可修复：`apiKey: "sk-xxx"` → `apiKey: "${API_KEY}"`
- 可修复：`http://` → `https://`
- `--fix --dry-run` → 只输出建议，不修改

**验收标准**：
- 对测试 fixture 运行 `--fix --dry-run` 输出正确的 diff
- ≥2 个测试

### 阶段 GA：GitHub Action（模块 1，依赖 SARIF）

#### 任务 GA1：创建 action.yml

**依赖**：任务 SF1（需要 `--format sarif`）

**验收标准**：

- README 中包含使用示例

### 阶段 W：白名单 + 预设（模块 5）

#### 任务 W1：实现 PL-04 白名单

**依赖**：无
**干什么**：
- 策略文件中支持 `exceptions` 字段
- 匹配逻辑：rule → tool → path → expires（过期日期检查）

**验收标准**：
- ≥3 个新增测试（正常豁免、过期豁免、路径豁免）
- `cargo check` 通过

#### 任务 W2：实现 PL-05 + PL-06 + policy init/check

**依赖**：任务 W1
**干什么**：
- 3 个内置预设：basic（只查 critical） / strict（全开） / enterprise（strict + 审计链）
- PL-06: 基线对比 — 将上次审计链结果作为基线，本次扫描对比输出 `+NEW` / `✓FIXED` / `=SAME`
- `hermes policy` → 生成默认 `.hermes-policy.json`
- `hermes policy --template dengbao` → 等保策略
- `hermes policy check <path>` → 加载策略文件，运行扫描，判断合规性（通过/不通过 + 不合规项列表）

**验收标准**：
- `--preset basic` 只输出 critical 级别
- `--preset strict` 全量输出
- `hermes policy` 生成有效 JSON
- `hermes policy check . --policy policy.json` 按策略判断合规
