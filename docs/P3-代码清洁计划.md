# 小的极致 — 代码清洁计划


> 日期：2026-06-02 | v0.3.1 维护版

---

## 一、需求文档

### 项目概要

| 项 | 内容 |
|:--|------|
| 项目名称 | Hermes v0.3.1 |
| 一句话 | 不改功能、不加依赖，把现有 30+ 文件约 3000 行代码抛光到无可挑剔 |
| 为什么要做 | P0-P2 快速开发累积了代码债——4 处重复、6 个文件有废代码标记、错误输出格式不统一、已安装的错误美化库没用 |

### 用户

| 用户类型 | 需求 |
|----------|------|
| 代码贡献者 | 看懂代码，不会在 4 个文件里看到同一段函数 |
| 调试/排错者 | 错误信息清晰，带上下文和颜色 |
| 维护者 | 没有 "#![allow(dead_code)]" 这种"先留着"的标记 |

### 核心功能（5 项）

| # | 做什么 | 怎么算做完 |
|:--:|------|------|
| 1 | discover_tools 去重 | 4 处重复 → 1 处公共函数，`cargo check` 通过 |
| 2 | color-eyre 激活 | 一行 `.install()`，所有错误自动带颜色 + 上下文 |
| 3 | eprintln! → 统一宏 | 约 30 处替换，输出格式一致 |
| 4 | 删除 dead_code | 6 个文件的 `#![allow(dead_code)]` 全部移除，零新增警告 |
| 5 | 计数函数合并 | count_audit + count_probe → 1 个泛型函数，`cargo test` 通过 |

### 特殊情况

| 情况 | 处理 |
|------|------|
| 改名后代码不编译 | 逐文件替换，每步 `cargo check` |
| clippy 新增警告 | 立即修，目标是 0 警告 |
| 改乱了恢复 | `git checkout` 回退 |

### 以后不做

- 不加新依赖（wiremock、criterion、cargo-dist）
- 不加新功能（CLI 不变）
- 不加测试（已有 171 项，够了）
- 不加文档

## 二、设计方案

### 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 公共 discover_tools | `src/probe/common.rs` | 4 个调用者全在 probe 模块下 |
| 错误宏 | 使用 `tracing::{error, warn}` + `color_eyre` | tracing 已安装，保持一致性 |
| 计数泛型 | trait `HasSeverity` | 两个类型共享 Severity 字段，用 trait 统一 |

### 变更清单

```
新增:
  src/probe/common.rs             # discover_tools 公共函数

修改:
  src/main.rs                     # +color_eyre 安装; 合并计数; eprintln → tracing
  src/probe/ssrf.rs               # 替换本地 discover_tools → common
  src/probe/traversal.rs          # 同上
  src/probe/redirect.rs           # 同上
  src/fuzz/engine.rs              # 替换本地 discover_tools → probe::common
  src/chain/hmac.rs               # eprintln → tracing
  src/audit/parser.rs             # 同上
  src/audit/scanner.rs            # 同上
  src/report/html.rs              # 同上

删除:
  #![allow(dead_code)] from 6 files
  fn count_audit / fn count_probe → 合并为 1 个
```

## 三、任务清单

**总共 5 个任务。**

### 任务 1：discover_tools 去重

**依赖**：无
**干什么**：
- 创建 `src/probe/common.rs`，放入公共 `pub async fn discover_tools(client, base)`
- 改 `src/probe/ssrf.rs` — 删除本地实现，调用 common
- 改 `src/probe/traversal.rs` — 同上
- 改 `src/probe/redirect.rs` — 同上
- 改 `src/fuzz/engine.rs` — 同上

**验收标准**：
- `cargo check` 通过
- `cargo test` 全项通过

---

### 任务 2：color-eyre 激活

**依赖**：无
**干什么**：
- `src/main.rs` 开头加一行 `color_eyre::install().unwrap();`

**验收标准**：
- 运行命令 `hermes audit bad-path` — 错误信息带蓝色箭头 + 红色高亮

---

### 任务 3：eprintln → tracing 宏

**依赖**：无
**干什么**：
- 全局搜索 `eprintln!` → 改为 `tracing::error!` 或 `tracing::warn!`
- 约 30 处，分布在 main.rs / chain/hmac.rs / audit/parser.rs 等

**验收标准**：
- `eprintln!` 在非测试代码中全部消除
- 错误信息可通过 `--verbose` 控制输出到 stderr

---

### 任务 4：删除 dead_code allow

**依赖**：所有任务完成后
**干什么**：
- 移除 6 个文件顶部的 `#![allow(dead_code)]`
- `cargo clippy` 检查 → 真死代码砍掉，真需要的保留

**验收标准**：
- clippy 0 warnings
- 所有测试通过

---

### 任务 5：计数函数合并

**依赖**：无
**干什么**：
- 合并 main.rs 中的 `fn count_audit` 和 `fn count_probe`
- 添加 trait `HasSeverity` 让 Finding 和 ProbeFinding 都能用

**验收标准**：
- 代码中只有一个计数函数
- `cargo test` 全项通过
